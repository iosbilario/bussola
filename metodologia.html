Honestidade é o moat.

Esta página existe porque a credibilidade da Bússola depende de você poder auditar cada número. Sem marketing: aqui está exatamente de onde vêm os dados, como o líquido é calculado e por que o ranking nunca é vendido.

01 De onde vem cada número

  • Câmbio comercial: PTAX do Banco Central (cotação de venda do fechamento mais recente), via API olinda — oficial e gratuita. Atualizado por uma GitHub Action em dias úteis, ~13h BRT, que reescreve data/cambio.json e commita. Cada commit é um snapshot datado.
  • Tarifas dos provedores: não existe API pública limpa para todas. Por isso data/tarifas.json é versionado à mão, com verificadoEm e fonte por provedor. Cada alteração vira um commit — o histórico de tarifas vive no git e é auditável linha a linha.
  • IOF: 0,38% sobre o valor convertido. É tributo, não taxa de provedor — incide igual para todos. Não é o que separa as opções.

02 Cada provedor corta de um jeito (e a UI mostra isso)

Usar um "spread genérico" para todos seria mentira. Existem dois modelos de custo reais, e a Bússola decompõe os dois de forma diferente:

comercial Wise, Husky. Convertem pelo câmbio comercial puro e cobram uma taxa explícita — fee percentual por faixa (Wise) ou taxa operacional com piso fixo (Husky). A fita mostra: Câmbio comercial → Convertido → Fee/Taxa.

spread Remessa, banco. Embutem a margem no próprio câmbio: o "câmbio aplicado" já vem pior que o comercial. A fita mostra: Câmbio comercial → Câmbio aplicado (−x%) → Convertido. Aqui o custo não é uma linha separada — está escondido no câmbio, e nós o expomos.

brutoComercial = v × c // a régua: o real puro comercial (Wise, Husky): convertido = v × c // câmbio comercial, sem spread custoFee = fee aplicado (faixa | tier) spread (Remessa, banco): efetivo = c × (1 − spreadPct) // spread piora o câmbio convertido = v × efetivo custoFee = 0 // o custo já está no câmbio iof = convertido × 0,0038 // igual pra todos tarifa = tarifaFixaBRL (se houver) liquido = convertido − iof − custoFee − tarifa mordida = brutoComercial − liquido

Tudo isto roda no seu navegador. Nenhum dado seu é enviado a lugar nenhum — não há backend, banco ou servidor.

03 A regra de neutralidade

O ranking é só matemática: ordenamos por liquido decrescente. Vence quem deixa mais real no seu bolso para aquele valor, moeda e câmbio — sempre, sem exceção. Se o banco entregasse mais, o banco viria em primeiro.

O líquido não conhece o link de afiliado. Zerar todos os links de afiliado produz exatamente o mesmo ranking — isso é testado automaticamente. Em valores baixos, a taxa fixa da Husky pode torná-la pior que a Remessa; o ranking reflete esse flip, mesmo que a Husky seja parceira.

04 Divulgação de afiliados

Alguns provedores têm programa de afiliado. Quando há link, a fita mostra "Abrir conta", e logo abaixo: "link de afiliado · não altera o ranking". Se o link ainda não existe, o botão fica desabilitado ("em breve"). O banco tradicional não tem afiliado — mostra apenas "conta via gerente / agência", sem botão.

Afiliados pagam as contas da Bússola. O que eles não compram é posição: a ordem é definida antes de qualquer link entrar na tela. Essa separação é o produto.
